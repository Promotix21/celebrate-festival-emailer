Design Adaptation Notes

Kept as-is
Adapted for email
Added for Klaviyo

Brand Identity

  • Same color palette: Navy #1a365d & Red #b91c22
  • Same celebrate-logo.png from repo
  • Same typography hierarchy & weight
  • Same content — every word preserved

Layout & Structure

  • Converted from div/flexbox to table-based layout
  • Max-width reduced from 900px to 600px (email standard)
  • Skewed/slanted CSS transforms replaced with straight rounded cards (email clients don't support transforms)
  • Step images moved to right column within table rows

Icons & Graphics

  • Lucide React SVG icons replaced with PNG image icons
  • Social media icons converted to PNG images
  • Trust bar icons are now inline PNG with text
  • All images have proper alt text for accessibility

CSS & Styling

  • Tailwind classes converted to inline CSS
  • CSS gradients preserved where possible with fallbacks
  • Hover effects removed (not supported in email)
  • Border-radius kept — supported in modern email clients

Responsiveness

  • Media queries at 620px breakpoint for mobile
  • Step images hidden on mobile (content preserved)
  • Trust bar stacks vertically on small screens
  • Font sizes adjust for mobile readability

Klaviyo Best Practices

  • Preheader text for inbox preview
  • Klaviyo unsubscribe tag: {% unsubscribe_url %}
  • View in browser tag: {% view_in_browser_url %}
  • Manage preferences tag: {% manage_preferences_url %}
  • VML fallback for Outlook background images
  • MSO conditional comments for Outlook rendering
  • role="presentation" on all tables
  • Apple Mail format-detection disabled

What Changed Visually

  • Skewed/angled step bars → straight rounded bars (CSS transforms unsupported in Outlook, Gmail)
  • Skewed images → straight rectangular rounded images
  • Dotted vertical timeline → dashed connectors between steps
  • Gradient overlays → simplified gradient with VML fallback

Production Checklist

  • Host all images on Klaviyo CDN or your domain
  • Replace placeholder social links with real URLs
  • Update consultation URL in CTA button
  • Update phone number
  • Test with Litmus / Email on Acid before sending
  • Set up A/B test for subject line in Klaviyo
